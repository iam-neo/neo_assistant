Overview

Welcome to Neo Assistant

A lightweight, rule-based local AI assistant designed to be fully modular and run entirely offline without requiring heavy external dependencies.

Neo takes text commands, interprets the user's intent via a straightforward parsing rule-engine initially, and uses an advanced Local LLM fallback mechanism to handle complex reasoning entirely on-device.

Core Features

🚀

System Triggers

Launch applications, websites, and create system folders instantly.

🧠

LLM Fallback

Phase 3 integration routes complex English and Nepali input to an offline Mistral model.

🇳🇵

Native NLP

Built-in tokenization parsing Romanized and Devanagari text seamlessly.

🔒

100% Offline

No API keys, internet connection, or telemetry required whatsoever.

Quickstart Setup

Because Neo Assistant prioritizes a zero-dependency architecture out of the box, getting started is practically instantaneous.

bash 
# 1. Clone the repository
git clone https://github.com/your-repo/neo-assistant.git

# 2. Navigate to project
cd neo-assistant

# 3. Start the interactive loop
python main.py

Configure your data/apps.json to map your custom desktop software natively into the parser's registry.

Deep Dive

Code Explanation: How "open youtube" Works

A complete line-by-line trace through every file involved when the user types "open youtube" into the Neo Assistant.

1

Input Capture

main.py → Line 39

The main loop waits for user input using Python's built-in input() function:

python — main.py
user_input = input("Neo> ")  # user_input = "open youtube"

The NeoAssistant.run() method contains a while self.running loop. Each iteration captures one command, parses it, and routes it.

2

Send to Parser

main.py → Line 46

The raw input is sent to the central parser which returns a structured (intent, data) tuple:

python — main.py
intent, data = parse_command("open youtube")
# Let's trace what happens inside parse_command()...
3

Text Normalization

parser.py → normalize_text()

Inside parse_command(), the first step normalizes the input — translating Nepali words and removing filler words:

python — utils/parser.py
normalized = normalize_text("open youtube")

# Inside normalize_text():
text = "open youtube".lower()     # → "open youtube" (already lowercase)

# Loops through NORMALIZATION_MAP looking for matches:
# "खोल"   → not found in text, skip
# "khola" → not found in text, skip
# "open"  → IS a key, but maps to "open" too — no change
# "please", "can you", etc. → not found, skip

return "open youtube"  # Nothing changed — already clean English

The normalization map uses regex word boundaries (\b) for English/Romanized words and direct string.replace() for Devanagari characters. Keys are sorted by length descending so multi-word phrases like "can you" are matched before single words.

4

Rule-Based Intent Matching

parser.py → parse_command()

The parser runs through a priority-ordered series of rule checks on the normalized text:

python — utils/parser.py
normalized = "open youtube"

# Priority 1: Exit check
if "exit" in normalized:         # ❌ "exit" not found, skip

# Priority 2: Create folder check
if "create folder" in normalized: # ❌ not found, skip

# Priority 3: Open check
if "open" in normalized:          # ✅ MATCH! Enter this block

    words = "open youtube".split()            # → ["open", "youtube"]
    target_words = [w for w in words if w != "open"]
                                              # → ["youtube"]
    target = " ".join(["youtube"])            # → "youtube"

    # Is target empty?
    if not target:                            # ❌ Not empty

    # Website or App? Check KNOWN_WEBSITES set
    # KNOWN_WEBSITES = {"youtube", "google", "github", ...}
    if "youtube" in KNOWN_WEBSITES:           # ✅ YES!
        return ("open_website", "youtube")    # 🎯 RESULT

Priority order matters! Exit is checked first (highest priority), then folder creation, then open commands. If nothing matches, the input falls through to the LLM fallback on line 116.

5

Intent Routing

main.py → Lines 49-52

Back in main.py, the returned tuple is matched to the correct executor:

python — main.py
intent, data = ("open_website", "youtube")

if intent == "open_app":          # ❌
    execute_open_app(data)
elif intent == "open_website":    # ✅ MATCH!
    execute_open_website("youtube")  # → Calls open_website.py
elif intent == "create_folder":   # ❌
    execute_create_folder(data)
6

Execute: Open YouTube

commands/open_website.py

The execute() function builds a full URL and opens it in the system's default browser:

python — commands/open_website.py
website_name = "youtube"

# Step 1: Build URL — does "youtube" contain a "."?
if "." not in "youtube":            # ✅ No dot found
    url = "https://www.youtube.com" # Auto-formats to full URL

# Step 2: Open in browser
success = webbrowser.open("https://www.youtube.com")

if success:
    print("[+] Successfully opened https://www.youtube.com")
    # 🌐 YouTube opens in your default browser!

The URL builder handles three formats: bare names like "youtube" become https://www.youtube.com, domains like "example.org" get https:// prefixed, and full URLs are used as-is.

Component Details

📥

main.py — Entry Point

Runs the interactive loop, captures input, calls parse_command(), and routes the (intent, data) tuple to the correct executor module.

🧠

parser.py — The Brain

normalize_text() translates Nepali/mixed input to English tokens. parse_command() extracts structured intents via priority-ordered rules.

🤖

llm.py — Smart Fallback

When rules can't determine intent, sends input to local Ollama Mistral with a strict system prompt. Temperature 0.0 ensures deterministic output.

⚙️

commands/ — Executors

Each module has an execute(data) function: open_app.py uses apps.json, open_website.py uses webbrowser, create_folder.py uses os.makedirs().

NLP Engine

Normalization Map Reference

The normalization engine translates mixed Nepali (Devanagari & Romanized) and English filler words into clean, structured English tokens.

Input Word Maps To Category
खोल / khola / kholopen🚀 Open / Launch
chalao / जाउ / jauopen🚀 Open / Launch
start / launchopen🚀 Open / Launch
बनाउ / banao / banacreate📁 Create
फोल्डरfolder📁 Create
nayacreate📁 Create
बन्द / bandaexit🛑 Exit / Stop
stop / quit / closeexit🛑 Exit / Stop
please / can you(removed)🗑️ Filler
gara / gar / ma(removed)🗑️ Filler
the / a / an(removed)🗑️ Filler

How matching works: English/Romanized words use regex word boundaries (\b) for precise whole-word matching. Devanagari characters (Unicode range \u0900–\u097F) use direct string.replace() since \b doesn't handle Devanagari cleanly. Keys are sorted by length (longest first) so multi-word phrases match before single words.
